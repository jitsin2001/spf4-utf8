Методика сравнения хэш-функций

Чтобы сравнить хэш-функции, для каждой
функции путем тестирования собираются некоторые 
оценочные характеристики, по которым и проводиться
сравнение.

Тестирование хэш-функции проводиться следующим образом:

Таблица размером hash-mod ячеек содержит счетчики
(hash-mod=2*1024).  Первоначально все счетчики =0.

Из файла wl.txt читаются слова (846 слов).
По хэш-образу каждого слова вычисляется адрес в таблице 
и соответствующий счетчик увеличивается на 1.

После обработки всех слов собирается статистика
по счетчикам.
Первое значение - число нулей, второе - число едениц,
и т.д. до 10. 11-ое значение - сумма других счетчиков.

Таким образом, на рис.1  первый столбец - число
пустых ячеек в таблице, второй - в ячейке одно слово,
Треттий столбец - в такое количество ячеек попало
по два слова (то бишь, одна коллизия), 
четвертый - две коллизии, и т.д.

Число свободных ячеек характеризует, насколько полно
использовано пространство таблицы. Чем оно меньше 
- тем лучше.
Второе число - число слов без коллизий - чем больше,
тем лучше.
