  Sedna as Windows service
 
  Sedna XML DBMS пока еще (на Март.2009) не умеет сама работать
виндовой службой. Обсуждение вопроса: http://article.gmane.org/gmane.text.xml.sedna/1101

  se_svc.f — программа-посредник, которая сама работает службой и управляет
Седной (запускает и останавливает) по ее правилам.

  make.cmd — сборка se_svc.exe (который следует положить в каталог sedna/bin/)
            Этот make можно запускать с указанием пути прямо
            из каталога sedna/bin/, тогда .exe прямо там и будет создан.     
            
            Для сборки требуется SP-Forth 4.21 (предполагается, что команда
            'spf' доступна через PATH и ведет на spf4.exe).
  
  Пример устновки службы (порядок аргументов важен):
  
    se_svc.exe -db 'auction auctionarc' -install    

        — будет создана служба с именем sedna, при запуске которой
        будет запущен se_gov и менеджеры указанных баз данных auction
        и auctionarc.
    
         
  Чтобы Седна нормально запустилась, необходимо, чтобы соответствующий
пользователь (по умолчанию, SYSTEM) имел R/W доступ на каталоги data и cfg
(и все файлы в них).

  Для проверки разрешений используйте следующие команды:

    sc qc sedna

    cacls sedna/data

Подставьте свой путь к каталогу data в соответствии с конфигом sedna/etc/sednaconf.xml 


  Рядом с бинарником se_svc.exe создается файлик se_svc.status — с информацией
по последнему запуску службы.
